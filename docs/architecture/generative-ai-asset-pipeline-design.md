# 생성 AI 에셋 파이프라인 설계 (Generative AI Asset Pipeline Design)

> **비코드 에셋(이미지/사운드/영상/3D)이 필요할 때, 최신 생성 AI 모델을 탐색하고 최적 도구를 안내하여 생성·출력하는 Guide-First 파이프라인**

**최종 수정**: 2026-04-06
**상태**: 설계 확정 (3+1 합의 검증 완료)
**상위 문서**: `PROJECT_CONSTITUTION.md` 제3조
**관련 ADR**: ADR-003 (생성 AI 에셋 파이프라인)

---

## 1. 개요

### 1.1 핵심 원칙

소프트웨어에는 코드만이 아니라 이미지, 사운드, 영상 등 **비코드 에셋**이 필요하다. 생성 AI 모델 생태계는 빠르게 변하므로, 고정된 도구가 아닌 **탐색 기반 최적 모델 선정**이 필요하다.

### 1.2 설계 원칙 (3+1 합의 결과)

| 원칙 | 설명 | 근거 |
|------|------|------|
| **Guide-First** | LLM이 도구/명령어/절차를 안내, 실행은 사용자가 판단 | Agent A 제안 + 3자 합의 |
| **4카테고리** | IMAGE, AUDIO, VIDEO, 3D만 분류. 세부 유형은 프롬프트에서 처리 | Agent A/C 합의 |
| **허브 우선** | 통합 API 허브(Replicate 등)를 기본 경로, 개별 API는 고급 옵션 | Agent C 제안 |
| **안전장치 최소 세트** | LLM 내장 거부 + 금지 키워드 + 라이선스 표기 + 비용 경고 | Agent B + Reviewer 절충 |
| **폴백 체인** | AI 모델 → 대체 AI → 비AI 도구(스톡/마켓) | Agent B/C 합의 |
| **반복 상한 2회** | 프롬프트 수정 2회, 불만족 시 모델 전환 또는 수동 안내 | Agent A 제안 |

### 1.3 파이프라인 위치

코드 구현과 **병렬로** 동작한다.

```
Phase 0: 검토 질문지 → 브리프
Phase 1: 3+1 합의 (아이디어 + 스택 + 에셋 요구 식별)
    │
    ├─── Phase 2-3: SDD/TDD (코드)
    │
    └─── 에셋 파이프라인 (비코드, Guide-First)
         → 탐색 → 안내 → 사용자 실행 → 검증 → 통합
```

## 2. 에셋 분류 (4카테고리)

### 2.1 카테고리

| 카테고리 | 범위 | 예시 |
|---------|------|------|
| **IMAGE** | 아이콘, UI, 일러스트, 다이어그램, 사진 | 앱 아이콘, 배경, 로고, 목업 |
| **AUDIO** | 효과음(SFX), 배경음악(BGM), 음성(TTS) | 클릭음, BGM, 나레이션 |
| **VIDEO** | 애니메이션, 실사 영상, 데모 | 로딩 애니메이션, 튜토리얼 |
| **3D** | 모델, 텍스처 | 게임 에셋, 제품 시각화 |

> TEXT/DATA는 코드 파이프라인(LLM 직접 생성)에서 처리. 에셋 파이프라인 범위 밖.

### 2.2 품질 수준

| 수준 | 설명 | 경로 |
|------|------|------|
| **Draft** | 프로토타입/목업. 빠르고 저렴 | Fast Track (LLM 즉시 추천) |
| **Production** | 최종 배포. 고품질 + 라이선스 확인 필수 | 정상 경로 (탐색 + 안내) |

## 3. 모델 탐색 (Model Discovery)

### 3.1 탐색 경로

```
┌─────────────────────────────────────────────┐
│  Layer 0: 비AI 대안 확인                      │
│  → 스톡 에셋(Unsplash, Freesound 등) 안내    │
│  → 에셋 마켓플레이스(Unity Store 등) 안내     │
│  → AI 생성보다 적합하면 여기서 종료           │
├─────────────────────────────────────────────┤
│  Layer 1: 통합 허브 기본 경로                  │
│  → Replicate, Together AI 등의 모델 허브 안내 │
│  → 카테고리별 인기/검증 모델 추천             │
│  → 대부분의 에셋은 이 경로로 충분             │
├─────────────────────────────────────────────┤
│  Layer 2: 개별 모델/서비스 (고급 옵션)         │
│  → 특정 요구에 최적화된 전문 서비스 안내       │
│  → WebSearch로 최신 모델/벤치마크 검색        │
│  → Layer 1이 부적합한 경우에만 사용           │
└─────────────────────────────────────────────┘
```

### 3.2 탐색 에이전트

메인 컨텍스트 오염 방지를 위해 **서브에이전트**가 탐색을 수행한다.

```
Agent(
  role="생성 AI 모델 탐색가",
  tools=[WebSearch, WebFetch],
  prompt="""
    에셋 카테고리: {category}
    세부 요구: {description}
    품질 수준: {draft/production}
    제약: {constraints}

    다음 순서로 탐색하세요:
    1. 비AI 대안 (스톡/마켓) 적합 여부
    2. 통합 허브(Replicate 등)의 적합 모델
    3. (필요 시) 전문 서비스/최신 모델

    각 추천에 대해 제공하세요:
    - 사용법 (URL, 명령어, API 호출법)
    - 예상 비용
    - 라이선스 조건
    - 품질 수준
  """
)
```

### 3.3 모델 평가 기준

| 기준 | 가중치 | 설명 |
|------|--------|------|
| **품질** | 높음 | 출력 품질, 해상도, 자연스러움 |
| **접근성** | 높음 | 사용 난이도, API 유무, 웹 UI |
| **라이선스** | 높음 (Production) | 상업적 사용 가능 여부 |
| **비용** | 중간 | 무료/유료, 호출당 비용 |
| **속도** | 중간 | 생성 소요 시간 |

## 4. Guide-First 실행 모델

### 4.1 LLM이 안내하는 것

LLM은 **직접 실행하지 않고**, 사용자가 실행할 수 있는 구체적 안내를 생성한다.

| 안내 유형 | 예시 |
|-----------|------|
| **웹 UI 안내** | "Replicate에서 flux-schnell 모델을 사용하세요. URL: ... 프롬프트: ..." |
| **API 호출 코드** | "다음 Python 코드로 생성할 수 있습니다: `requests.post(...)` " |
| **CLI 명령어** | "`comfyui --model sd-xl --prompt '...'` 으로 로컬 실행" |
| **설치 가이드** | "로컬 실행을 위해: `pip install audiocraft` → 다음 코드 실행" |

### 4.2 실행 흐름

```
에셋 요구 식별
    │
    ▼
탐색 에이전트: 최적 도구/모델 추천 + 사용법 안내
    │
    ▼
안전장치 체크:
  ✓ 라이선스 정보 표기
  ✓ 예상 비용 경고 (유료 시)
  ✓ 금지 키워드 체크 (프롬프트)
    │
    ▼
사용자에게 안내 전달:
  "다음 방법으로 {에셋}을 생성할 수 있습니다:
   [방법 1] {웹 UI} — 가장 쉬움
   [방법 2] {API 코드} — 자동화 가능
   프롬프트 제안: '{최적화된 프롬프트}'
   예상 비용: {비용}
   라이선스: {조건}"
    │
    ▼
사용자 실행 → 결과 확인
    │
    ├── 만족 → 저장 + MANIFEST 기록
    │
    └── 불만족 → 프롬프트 수정 안내 (최대 2회)
        → 2회 후에도 불만족 → 모델 전환 안내 또는 수동 제작 권장
```

## 5. 안전장치

### 5.1 최소 안전 세트

| 장치 | 방식 | 설명 |
|------|------|------|
| **콘텐츠 안전** | LLM 내장 거부 + 금지 키워드 목록 | 부적절 프롬프트 사전 차단 |
| **라이선스 표기** | 모든 안내에 라이선스 정보 필수 포함 | 사용자가 판단할 수 있도록 정보 제공 |
| **비용 경고** | 유료 서비스 안내 시 예상 비용 표기 | "이 API 호출 1회당 약 $0.04입니다" |
| **API 키 보안** | 환경변수 참조 안내, 프롬프트 내 키 노출 금지 | 헌법 제8조 준수 |

### 5.2 Production 에셋 추가 체크

Production 에셋 안내 시 반드시 포함:

```
⚠️ Production 에셋 체크리스트:
  □ 라이선스가 상업적 사용을 허용하는가?
  □ 저작자 표시(attribution) 의무가 있는가?
  □ 생성 모델의 학습 데이터에 저작권 이슈가 있는가?
  □ 출력물의 저작권 귀속은 누구에게 있는가?
```

## 6. 폴백 체인

```
1차: 추천 AI 모델로 생성 시도
    │
    ├── 성공 → 완료
    │
    └── 실패 (모델 불가용, 품질 부족)
        │
        ▼
2차: 대체 AI 모델 안내
    │
    ├── 성공 → 완료
    │
    └── 실패
        │
        ▼
3차: 비AI 대안 안내
    → 스톡 에셋 사이트 (Unsplash, Freesound, Pixabay)
    → 에셋 마켓플레이스 (Unity Store, Envato)
    → 외주 플랫폼 (Fiverr, 크몽)
    → 수동 제작 도구 안내 (Figma, Audacity, Blender)
```

## 7. 출력 + 프로젝트 통합

### 7.1 저장 구조

```
assets/
├── images/          # IMAGE 에셋
├── audio/           # AUDIO 에셋
├── video/           # VIDEO 에셋
├── models/          # 3D 에셋
└── manifest.json    # 에셋 매니페스트 (기계 판독)
```

### 7.2 매니페스트 스키마 (manifest.json)

```json
{
  "assets": [
    {
      "file": "images/logo.png",
      "category": "IMAGE",
      "quality": "production",
      "model": "DALL-E 3",
      "prompt": "미니멀 로고...",
      "license": "상업적 허용",
      "cost": "$0.04",
      "created": "2026-04-06",
      "notes": ""
    }
  ]
}
```

## 8. Phase 1 합의에서의 에셋 식별

Phase 1(아이디어 합의) 시 에이전트가 비코드 에셋 요구를 자동 식별한다.

```
에이전트 프롬프트에 추가:

"이 아이디어에 코드 이외의 에셋(이미지, 사운드, 영상, 3D)이
 필요한지 식별하세요. 필요한 경우 각 에셋의:
 - 카테고리 (IMAGE/AUDIO/VIDEO/3D)
 - 세부 설명
 - 품질 수준 (Draft/Production)
 을 합의 보고서에 포함하세요."
```

## 9. 적용 시나리오

```
사용자: "게임에 8bit 배경음악과 픽셀 아트 아이콘이 필요해"

Phase 1 합의 → 에셋 식별:
  IMAGE(픽셀 아트 아이콘, Draft) + AUDIO(8bit BGM, Production)

에셋 파이프라인:

  [IMAGE - Draft - Fast Track]
  탐색: Layer 0 → 비AI: pixel art 스톡은 범용적이지 않음
        Layer 1 → Replicate: SDXL + pixel art LoRA 추천
  안내: "Replicate에서 SDXL pixel-art 모델을 사용하세요.
        프롬프트: '16x16 pixel art game icon, {object}, transparent bg'
        비용: 약 $0.01/이미지"
  사용자 실행 → 저장

  [AUDIO - Production - 정상 경로]
  탐색: Layer 0 → 비AI: Freesound에 8bit 무료 SFX 있음 (BGM은 부족)
        Layer 1 → Suno: 8bit/chiptune 장르 지원, 상업적 라이선스 (Pro)
  안내: "Suno(suno.ai)에서 8bit chiptune BGM을 생성하세요.
        프롬프트: '8-bit chiptune adventure game BGM, upbeat, 120 BPM'
        비용: Pro 구독 $10/월
        ⚠️ Production 체크: 라이선스=Pro 구독 시 상업적 허용"
  사용자 실행 → 품질 확인 → manifest.json 기록
```

## 10. 3+1 합의 검증 결과 요약

### 핵심 설계 변경

| 변경 항목 | 초안 | 최종 | 근거 |
|-----------|------|------|------|
| 실행 모델 | API 직접 호출 전제 | **Guide-First (안내 중심)** | Claude가 직접 실행 불가, 사용자 판단 존중 |
| 에셋 분류 | 12유형 | **4카테고리** (IMAGE/AUDIO/VIDEO/3D) | TEXT/DATA는 코드 파이프라인 흡수 |
| 탐색 경로 | 3단계 (내부→웹→허브) | **비AI 우선 → 허브 → 개별 모델** | 비AI가 더 적합한 경우 존재 |
| Production 합의 | 3+1 합의 | **LLM 안내 + 사용자 판단** | 에셋은 재생성 비용 낮음, 합의 과도 |
| 안전장치 | 없음 | **LLM 거부 + 금지 키워드 + 라이선스 표기 + 비용 경고** | 최소 세트 |
| 반복 | 최대 3회 | **최대 2회** | 비용 절감 |
| 폴백 | 없음 | **AI → 대체 AI → 비AI 도구** | 모델 불가용 대응 |
| MANIFEST | Markdown | **JSON** | 기계 판독 가능 |
