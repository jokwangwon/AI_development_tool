## 1. 게임 개요

### 1.1 한 줄 요약

> 버튼 몇 개로 환경만 바꿔주고, 테이블 위 미니어처 주민들이 알아서 살아가는 모습을 지켜보는 관찰형 힐링 게임

### 1.2 핵심 키워드

| 항목 | 값 |
|------|-----|
| **장르** | 관찰형 힐링 시뮬레이션 / 갓 게임 |
| **플랫폼** | PC (Steam, itch.io) |
| **엔진** | Unity 6 LTS + URP |
| **시점** | 고정 앵글 3D (45도, 90도 스냅 회전) |
| **타겟 유저** | 힐링 게임 선호, 캐주얼 게이머 |
| **플레이 시간** | 회당 10~30분, 장기 플레이 |
| **ESRB** | E (Everyone) |

### 1.3 컨셉 아트 레퍼런스

- YouTube 미니어처 영상: 거대한 손이 양동이로 물을 뿌리면 미니어처 세상에 비가 내리는 장면
- 덕 시뮬레이터: 최소 조작으로 관찰하는 감성
- Townscaper: 아기자기한 미니어처 마을 미학
- 자연 다큐멘터리: NPC가 신을 인식하지 않고 자연스럽게 행동

### 1.4 핵심 설계 철학

```
1. 최소 조작 (Minimal Interaction)
   → 플레이어는 환경 토글 버튼과 카메라만 조작한다
   → NPC에 대한 직접 상호작용은 없다

2. 자율 생활 (Autonomous Living)
   → 주민들은 스스로 판단하고 행동한다
   → 환경 변화에 자연스럽게 적응한다
   → 신의 존재를 인식하지 않는다

3. 관찰의 재미 (Joy of Observation)
   → "보는 것만으로 재미있는" 상태를 최우선으로 추구
   → 자연 다큐멘터리를 보는 듯한 감성
   → 예측 불가능한 NPC 행동이 관찰의 핵심 동력

4. 미니어처 미학 (Miniature Aesthetics)
   → 테이블 위 디오라마 세상
   → 틸트 시프트 효과로 미니어처 느낌 강조
   → 현실 오브젝트(연필, 지우개 등)가 스케일 대비 제공
```

## 2. 게임 메카닉

### 2.1 플레이어 조작

#### 환경 토글 버튼 (핵심 메카닉)

화면 상단에 환경 버튼이 배치되며, 클릭으로 ON/OFF 토글한다.

| 버튼 | 효과 | 주민 반응 예시 |
|------|------|---------------|
| ☔ 비 | 디오라마에 비가 내림. 작물 성장 촉진, 물웅덩이 생성 | 밭에 물주기, 우산 쓰기, 실내로 들어감 |
| ☀️ 햇빛 | 강한 햇빛. 작물 건조, 밝은 분위기 | 빨래 널기, 밖에서 놀기, 일광욕 |
| 🌬️ 바람 | 바람이 붊. 풍차 작동, 낙엽 날림 | 모자 잡기, 풍차 근처 작업, 연 날리기 |
| ❄️ 겨울 | 눈이 내리고 추워짐. 수면 동결 | 모닥불 모이기, 눈사람 만들기, 실내 활동 증가 |

**토글 규칙:**
- 한 번에 하나만 활성화 가능 (비 + 겨울 동시 불가)
- 아무것도 활성화하지 않으면 기본 날씨 (맑음)
- ON 후 일정 시간이 지나면 서서히 효과 적용
- OFF 후 서서히 원래 상태로 복귀

#### 카메라 조작

| 조작 | 기능 |
|------|------|
| 마우스 스크롤 | 줌 인/아웃 |
| Q / E 키 | 90도 단위 회전 (스냅) |
| 마우스 드래그 (중앙 버튼) | 패닝 (화면 이동) |

**카메라 제한:**
- 고정 45도 앵글 (상하 각도 변경 불가)
- 디오라마 경계 밖으로 이동 불가
- 최대 줌 인: NPC 한 명이 화면의 1/3 차지
- 최대 줌 아웃: 디오라마 전체가 보이는 수준

#### 그 외 조작

| 조작 | 기능 |
|------|------|
| NPC 클릭 | NPC 정보 패널 표시 (이름, 현재 행동, 상태) — 관찰 전용 |
| 건물 클릭 | 건물 정보 표시 (이름, 거주자, 용도) — 관찰 전용 |
| ESC | 메뉴 (저장, 설정, 도감) |
| Tab | 도감 열기 |
| Space | 시간 배속 토글 (1x / 2x / 4x) |

### 2.2 게임 루프

```
┌─────────────────────────────────────────┐
│           메인 게임 루프                  │
│                                         │
│  ① 디오라마를 바라본다 (관찰)            │
│       │                                 │
│  ② 주민들이 자율적으로 생활한다           │
│       │                                 │
│  ③ 환경 버튼을 눌러본다 (유일한 조작)     │
│       │                                 │
│  ④ 주민들이 환경에 적응한다              │
│       │ (새 행동 패턴 발견 → 도감 등록)   │
│       │                                 │
│  ⑤ 시간이 흐르며 마을이 성장한다          │
│       │ (인구 증가, 건물 건설, 영역 확장)  │
│       │                                 │
│  ⑥ 성장 조건 충족 → 새 요소 해금          │
│       │ (새 디오라마, 새 환경 도구)        │
│       │                                 │
│  └──→ ①로 돌아감                        │
└─────────────────────────────────────────┘
```

### 2.3 시간 시스템

| 단위 | 게임 내 | 실제 시간 |
|------|---------|----------|
| 1시간 | 게임 1시간 | 실제 1분 |
| 1일 | 게임 24시간 | 실제 24분 |
| 1계절 | 게임 30일 | 실제 ~12시간 |

**시간대별 변화:**
- **새벽 (5-7시)**: 주민 기상, 기지개
- **오전 (7-12시)**: 농사, 건설 활동
- **오후 (12-17시)**: 탐험, 사교 활동
- **저녁 (17-20시)**: 귀가, 저녁 식사
- **밤 (20-5시)**: 취침 (모닥불/별 관찰 이벤트)

### 2.4 마을 성장 시스템

주민들이 자율적으로 마을을 성장시킨다. 플레이어가 적절한 환경을 제공하면 성장 속도가 빨라진다.

| 단계 | 상태 | 인구 | 건물 | 해금 조건 |
|------|------|------|------|-----------|
| Lv.1 정착 | 텐트 2~3개, 모닥불 | 3~5명 | 0 | 시작 |
| Lv.2 마을 | 오두막, 밭 | 5~8명 | 3~5개 | 첫 수확 성공 |
| Lv.3 발전 | 다양한 건물, 창고 | 8~12명 | 5~10개 | 건물 5개 이상 |
| Lv.4 번영 | 광장, 시장 | 12~15명 | 10개+ | 인구 12명 이상 |

**성장 메카닉:**
- 적절한 비 → 작물 성장 → 식량 증가 → 인구 증가
- 적절한 햇빛 → 건조/건설 용이 → 건물 건설 촉진
- 적절한 바람 → 풍차 작동 → 곡물 가공 → 발전 가속
- 환경을 방치하면 → 성장이 느려지지만 멈추지는 않음 (자율 생존)

## 3. NPC 시스템

### 3.1 NPC 설계 원칙

```
1. 완전 자율: 플레이어 개입 없이도 생활 가능
2. 환경 반응: 날씨/시간에 따라 행동 변화
3. 자연스러움: 타이밍 랜덤화, 경로 다양화, Idle 변형
4. 신 비인식: NPC는 환경 변화를 자연현상으로 인식
```

### 3.2 NPC 속성

```
NPCData:
  - id: string (고유 식별자)
  - name: string (이름)
  - job: enum (농부, 건축가, 탐험가, 요리사, ...)
  - personality: enum (부지런, 느긋, 호기심, 소심, ...)
  - schedule: Schedule (일과 스케줄)
  - mood: float (0~100, 기분)
  - hunger: float (0~100, 배고픔)
  - energy: float (0~100, 에너지)
```

### 3.3 NPC 행동 — FSM (유한 상태 머신)

```
                    ┌─────────────┐
            ┌──────→│   Sleeping   │◄──── 밤, 에너지 0
            │       └──────┬──────┘
            │              │ 아침 + 에너지 회복
            │              ▼
            │       ┌─────────────┐
            │  ┌───→│    Idle      │←─── 할 일 없음
            │  │    └──┬───┬───┬──┘
            │  │       │   │   │
            │  │       ▼   │   ▼
            │  │  ┌──────┐ │ ┌──────────┐
            │  │  │ Work │ │ │ Explore  │
            │  │  │ 농사 │ │ │ 탐험     │
            │  │  │ 건설 │ │ │ 발견     │
            │  │  └──┬───┘ │ └────┬─────┘
            │  │     │     │      │
            │  │     ▼     ▼      │
            │  │  ┌──────────┐    │
            │  └──│   Eat    │◄───┘
            │     │  식사    │
            │     └────┬─────┘
            │          │
            │          ▼
            │     ┌──────────┐
            └─────│  Rest    │
                  │  휴식    │
                  └──────────┘
```

### 3.4 환경에 따른 NPC 행동 변화

| 환경 | 농부 | 건축가 | 탐험가 |
|------|------|--------|--------|
| **비** | 밭에서 기뻐함, 수확 | 작업 중단, 실내 이동 | 비를 맞으며 걸음 |
| **햇빛** | 빨래 널기, 건조 작업 | 건설 활발 | 먼 곳 탐험 |
| **바람** | 풍차 확인, 곡물 가공 | 높은 건설 주의 | 연 날리기 |
| **겨울** | 저장 식량 꺼냄 | 실내 수리 | 실내 탐색 |
| **없음(맑음)** | 기본 농사 사이클 | 기본 건설 | 근거리 탐험 |

### 3.5 NPC 자연스러움 요소

| 요소 | 구현 방법 |
|------|----------|
| 타이밍 랜덤화 | 스케줄 시간 ±10분 랜덤 오프셋 |
| 경로 다양화 | 같은 목적지 2~3개 경로 중 랜덤 |
| Idle 변형 | 가만히 서기, 주변 둘러보기, 기지개 (3종+) |
| 도착 연출 | 건물 앞 잠시 멈춤 → 문 열림 → 진입 |
| 속도 변화 | 아침 빠르게, 저녁 느리게 |
| 근접 반응 | NPC끼리 가까이 지나가면 고개 끄덕이기 |
| 성격 반영 | 부지런한 NPC는 일찍 시작, 느긋한 NPC는 여유 |

## 4. 디오라마 (맵) 시스템

### 4.1 디오라마 구조

```
┌─ 현실 세계 (테이블) ──────────────────────────┐
│                                              │
│  ┌─ 디오라마 (미니어처 세상) ──────────────┐   │
│  │                                       │   │
│  │  지형 + 건물 + NPC + 자연물            │   │
│  │  (약 30m × 30m 실제 Unity 단위)        │   │
│  │                                       │   │
│  └───────────────────────────────────────┘   │
│                                              │
│  주변: 연필, 지우개, 책, 머그컵 (스케일 대비)  │
│                                              │
└──────────────────────────────────────────────┘
```

### 4.2 MVP 디오라마 — "초원 정착지"

**테마**: 원시 시대 초원 마을
**크기**: 30m × 30m (Unity 단위)
**지형**: 평탄한 초원 + 작은 언덕 + 개울

**배치 요소:**
- 주거지 영역: 텐트/오두막 건설 구역
- 농경지 영역: 밭, 과수원 구역
- 자연 영역: 숲, 개울, 바위
- 탐험 영역: 맵 가장자리 미지 영역

**주변 현실 오브젝트:**
- 나무 테이블 표면 (디오라마 받침)
- 거대한 연필 (1~2개)
- 지우개
- 동전 (스케일 대비)

### 4.3 시리즈 확장 디오라마 (향후)

| 시리즈 | 디오라마 예시 | 고유 환경 도구 |
|--------|-------------|---------------|
| Vol.1 원시 시대 | 초원, 강가, 동굴 근처 | 물방울, 횃불, 돌, 나뭇잎 |
| Vol.2 농경 시대 | 논밭, 마을, 목장 | 양동이, 전등, 부채, 씨앗 |
| Vol.3 중세 시대 | 성, 시장, 항구 | 물뿌리개, 랜턴, 톱니바퀴 |
| Vol.4 근대 시대 | 도시, 공장, 항구 | 호스, 서치라이트, 설계도 |

## 5. 도감 시스템

### 5.1 도감 카테고리

| 카테고리 | 수집 대상 | 발견 조건 예시 |
|---------|----------|---------------|
| **주민 도감** | NPC 개인별 프로필 | NPC 클릭으로 관찰 |
| **행동 도감** | NPC의 특정 행동 | 특정 환경에서 발생하는 희귀 행동 목격 |
| **건물 도감** | 건설된 건물 종류 | 마을 성장에 따라 등장 |
| **자연 도감** | 동식물, 자연 현상 | 특정 환경 조합에서 발생 |
| **이벤트 도감** | 특별 이벤트 | 특정 조건 조합 시 발생 |

### 5.2 희귀 행동 예시 (발견의 재미)

| 행동 | 발동 조건 | 희귀도 |
|------|----------|--------|
| NPC가 눈사람 만들기 | 겨울 + NPC 2명 이상 근접 | ★★☆ |
| NPC가 무지개 보고 감탄 | 비 직후 햇빛 ON | ★★★ |
| NPC가 별똥별에 소원 | 밤 + 맑음 + 랜덤 확률 | ★★★★ |
| 모든 NPC가 춤추기 | 풍작 수확 직후 | ★★★ |
| NPC 커플 산책 | 호감도 높은 NPC 2명 + 저녁 | ★★☆ |

## 6. 비주얼 & 오디오

### 6.1 아트 스타일

- **로우폴리 파스텔톤**: 둥글고 부드러운 형태, 파스텔 색감
- **틸트 시프트 효과**: URP Post Processing으로 화면 상하 블러 → 미니어처 느낌
- **스케일 대비**: 디오라마 주변 거대한 현실 오브젝트 (연필, 동전)
- **따뜻한 조명**: 부드러운 ambient light + 따뜻한 색온도
- **NPC 크기**: 디오라마 내에서 약 0.3~0.5m (작고 귀여운 느낌)

### 6.2 시각 효과

| 효과 | 구현 방식 |
|------|----------|
| 틸트 시프트 | URP Depth of Field (Bokeh) |
| 비 | Particle System (떨어지는 물방울) |
| 눈 | Particle System (눈송이) |
| 바람 | 풀/나무 흔들림 (Shader), 낙엽 파티클 |
| 햇빛 | Directional Light 강도 변화, 렌즈 플레어 |
| 낮/밤 | Skybox 색상 변화 + Light 색온도/강도 변화 |

### 6.3 오디오

| 상황 | 사운드 |
|------|--------|
| 기본 BGM | 잔잔한 피아노/어쿠스틱 (루프) |
| 비 | 빗소리 ambient |
| 바람 | 바람 소리 ambient |
| 겨울 | 조용한 ambient + 바삭바삭 발자국 |
| NPC 활동 | 망치질, 밭 가는 소리 등 (거리 기반 볼륨) |

## 8. 저장 시스템

### 8.1 세이브 데이터 구조

```
SaveData:
  version: int          // 세이브 파일 버전 (마이그레이션용)
  timestamp: DateTime   // 저장 시점
  
  gameTime: GameTime    // 게임 내 시간/날짜
  weatherState: Weather // 현재 환경 상태
  
  village:
    level: int          // 마을 레벨
    buildings: List<BuildingData>
    
  npcs: List<NPCData>   // NPC 상태 전체
  
  relationships: List<RelationshipData>  // 별도 테이블로 분리
  
  collection:
    discoveredNPCs: List<string>
    discoveredBehaviors: List<string>
    discoveredBuildings: List<string>
    discoveredEvents: List<string>
```

### 8.2 저장 규칙

- **자동 저장**: 게임 내 1일(24분) 종료 시
- **수동 저장**: ESC 메뉴에서 가능
- **임시 파일 → 원본 덮어쓰기** (atomic write)
- **백업 파일 1개 유지** (이전 세이브)

## 10. 개발 로드맵

### Phase 0: 학습 기초 (2~3주)

- [ ] Unity 에디터 기본 조작 학습
- [ ] C# 기본 문법 학습
- [ ] Unity "Roll-a-Ball" 튜토리얼 완수
- [ ] NavMesh 기초 튜토리얼 1개

### Phase 1: MVP 프로토타입 — "걸어다니는 미니어처" (3~4주)

**목표**: 디오라마 위에서 NPC가 걸어다니고, 환경 버튼이 동작한다

- [ ] URP 프로젝트 생성 + 기본 설정
- [ ] 고정 앵글 카메라 (45도, 줌, 회전)
- [ ] 평면 디오라마 맵 + 테이블 배경
- [ ] 틸트 시프트 포스트 프로세싱
- [ ] NPC 1명 NavMesh 이동 (FSM 3상태: Sleep → Walk → Idle)
- [ ] TimeManager (낮/밤 사이클)
- [ ] 환경 버튼 UI (비 ON/OFF) + 파티클

**✅ 검증**: "미니어처 세상을 바라보는 것이 시각적으로 매력적인가?"

### Phase 2: 자율 생활 — "살아있는 마을" (4~5주)

**목표**: NPC 5명이 자율적으로 일과를 수행하고, 환경에 반응한다

- [ ] NPC 5명 확장 (각각 다른 직업/성격)
- [ ] FSM 확장 (6상태: Sleep, Idle, Work, Eat, Explore, Rest)
- [ ] 환경 버튼 4종 전체 구현 (비, 햇빛, 바람, 겨울)
- [ ] NPC 환경 반응 행동 (상태별 분기)
- [ ] NPC 자연스러움 요소 (랜덤화, 경로 변형, Idle 변형)
- [ ] NPC 클릭 시 정보 패널
- [ ] 기본 마을 성장 (Lv.1 → Lv.2)

**✅ 검증**: "NPC 행동을 관찰하는 것이 재미있는가? 10분 이상 보고 싶은가?"

### Phase 3: 수집과 성장 — "도감과 발견" (4~5주)

**목표**: 도감 시스템으로 장기 동기부여, 마을 성장 전체 구현

- [ ] 도감 시스템 (행동 도감, 주민 도감)
- [ ] 희귀 행동 10종 이상
- [ ] 마을 성장 Lv.1~Lv.4 전체
- [ ] 저장/불러오기 시스템
- [ ] 건물 종류 8~10가지
- [ ] 시간 배속 기능

**✅ 검증**: "도감을 채우고 싶은 동기가 생기는가?"

### Phase 4: 폴리싱 — "출시 준비" (3~4주)

- [ ] 에셋 교체 (로우폴리 아트 적용)
- [ ] 사운드/BGM 추가
- [ ] 밸런싱 (성장 속도, 행동 빈도)
- [ ] 버그 수정
- [ ] 메인 메뉴 + 온보딩
- [ ] itch.io 빌드 및 업로드

**총 예상 기간: 16~21주 (약 4~5개월)**
