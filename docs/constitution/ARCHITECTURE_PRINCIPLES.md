# 아키텍처 원칙 (Architecture Principles)

> **AI 자동화 개발 도구의 아키텍처 설계 10대 원칙**

**최종 수정**: 2026-04-06
**상위 문서**: `PROJECT_CONSTITUTION.md` 제3조, 제6조

---

## 원칙 1: 하네스 중심 설계 (Harness-Centric Design)

시스템의 핵심은 LLM 모델이 아니라 **하네스**이다. 하네스가 도구, 메모리, 권한, 오케스트레이션, 안전 경계, 검증 루프를 제공한다.

- 모든 에이전트 기능은 하네스를 통해 제공된다
- 하네스 없는 모델 호출은 금지한다
- 하네스의 각 구성요소는 교체/제거 가능해야 한다

## 원칙 2: 가이드와 센서의 균형 (Balance Guides & Sensors)

**가이드(Feedforward)** 만으로 규칙을 인코딩하면 검증 없이 규칙이 무시될 수 있다.
**센서(Feedback)** 만으로 사후 검증하면 에이전트가 같은 실수를 반복한다.

- 가이드: CLAUDE.md, 설계 문서, 프롬프트 템플릿
- 센서: 린터, 테스트, 코드 리뷰 에이전트, 타입 체커
- 두 가지를 반드시 함께 설계한다

## 원칙 3: 계산적 검증 우선 (Computational Over Inferential)

결정적(deterministic)이고 빠른 계산적 검증을 우선한다. 추론적 검증(LLM 기반)은 판단이 필요한 모호한 상황에서만 사용한다.

| 우선순위 | 검증 방법 | 예시 |
|---------|----------|------|
| 1 | 계산적 | 린터, 타입 체커, 유닛 테스트 |
| 2 | 추론적 | LLM 코드 리뷰, 의미 분석 |

## 원칙 4: 관심사 분리 (Separation of Concerns)

각 모듈/에이전트는 단일 책임을 가진다.

- 생성과 평가를 분리한다 (생성 에이전트 ≠ 평가 에이전트)
- 오케스트레이션과 실행을 분리한다
- 데이터 접근과 비즈니스 로직을 분리한다

## 원칙 5: 파일 기반 에이전트 간 통신 (File-Based Agent Communication)

에이전트 간 통신은 구조화된 파일(아티팩트)을 통한다.

- 대화 기반 전달 대신 파일 기반 핸드오프를 사용한다
- 각 핸드오프 지점에서 다음 에이전트가 이전 에이전트의 출력을 검증한다
- 체크포인트가 명확히 정의되어야 한다

## 원칙 6: 독립적 검증 (Independent Verification)

에이전트 자체 평가에 의존하지 않는다.

- 생성 에이전트는 자신의 출력을 과대평가하는 경향이 있다
- 독립적인 검토 에이전트를 반드시 사용한다
- 3+1 합의 프로토콜을 통해 다관점 검증을 수행한다

## 원칙 7: 점진적 안전장치 (Graduated Guardrails)

안전장치를 다층으로 운영한다.

```
Level 0: 권한 경계 (파일/명령 접근 제한)
Level 1: 검증 체크 (린터, 타입 체커)
Level 2: 아키텍처 제약 (구조적 규칙 강제)
Level 3: 속도 제한 (과도한 실행 방지)
Level 4: 멀티 에이전트 합의 (중대 결정 검증)
```

## 원칙 8: 관찰 가능성 (Observability)

에이전트의 행동과 의사결정 과정이 추적 가능해야 한다.

- 모든 에이전트 액션은 로그로 남긴다
- 합의 보고서는 영구 저장한다
- 실패한 검증의 원인을 추적할 수 있어야 한다

## 원칙 9: 하네스 경량화 (Harness Minimalism)

하네스의 모든 구성요소는 **모델이 할 수 없는 것**에 대한 가정을 인코딩한다. 모델이 발전하면 더 이상 필요 없는 하네스 요소를 제거한다.

- 정기적으로 하네스 가정을 스트레스 테스트한다
- 불필요한 스캐폴딩은 제거한다
- 과도한 하네스는 개발 속도를 저해한다

## 원칙 10: 문서화된 아키텍처 결정 (Documented Decisions)

모든 아키텍처 결정은 ADR(Architecture Decision Record)로 기록한다.

- 결정의 맥락, 선택지, 근거를 포함한다
- 대체된 결정은 "Superseded" 상태로 보존한다
- 3+1 합의 결과가 있으면 보고서를 ADR에 첨부한다
