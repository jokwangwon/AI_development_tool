# ADR-001: 자동화 검토 질문지 설계

**상태**: 승인 (3+1 합의 완료)
**날짜**: 2026-04-06
**의사결정자**: 3+1 에이전트 합의

---

## 맥락 (Context)

사용자가 아이디어를 제시하면 3+1 에이전트가 분석하지만, 아이디어의 완성도가 낮으면 에이전트 분석 품질도 떨어진다. 아이디어와 에이전트 사이에 구조화 계층이 필요하다.

## 결정 (Decision)

**하이브리드 경량 자동화 검토 질문지**를 Layer 0.5 가이드로 도입한다.

핵심 설계:
- LLM-First 구조화 (자유 텍스트 → 자동 브리프)
- 최소 질문 (필수 3개 + LLM 동적 2개, 상한 5개)
- primary + secondary 복합 유형 분류 (4유형)
- 바이패스 경로 (장애 시 원문 직접 전달)
- 통과/미달 이진 완성도 판정

## 3+1 에이전트 합의 결과

| 에이전트 | 의견 | 핵심 근거 |
|---------|------|----------|
| Agent A (구현) | 조건부 동의 | 구현 가능하나 Smart Extraction 검증 필수 |
| Agent B (품질) | 조건부 동의 | 바이패스 경로 + 추출 확인 + 문서 동기화 필수 |
| Agent C (대안) | 대폭 경량화 요구 | 6유형/질문 매트릭스가 과도, LLM-First 권장 |
| **Reviewer** | **하이브리드 채택** | **골격 유지, 살 빼기, 안전망 추가** |

### CRITICAL 합의 사항 (3자 일치)
1. 복합 유형 미지원 → primary + secondary 구조
2. Smart Extraction 검증 부재 → 사용자 확인 단계 필수
3. Phase 0 SPOF → 바이패스 경로 설계

## 근거 (Rationale)

- Agent C의 "LLM-First 완전 자동화"는 누락 감지 정확도에 의존하여 순환 문제 발생
- Agent A/B의 "기존 구조 유지"는 질문 수 과다와 유지보수 부담
- 하이브리드: LLM 구조화 + 고정 필수 3개 + 동적 2개가 최적 균형점

## 결과 (Consequences)

- 긍정적: 사용자 마찰 최소화 (최대 5개 질문), 유지보수 부담 경감
- 긍정적: 안전장치 확보 (바이패스, 추출 확인, 복합 유형)
- 부정적: LLM 동적 질문의 품질이 프롬프트에 의존
- 주의: 분기마다 질문지 가치를 검증하고, 불필요 시 제거 (하네스 경량화)
