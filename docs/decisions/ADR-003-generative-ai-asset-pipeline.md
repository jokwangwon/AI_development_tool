# ADR-003: 생성 AI 에셋 파이프라인

**상태**: 승인 (3+1 합의 완료)
**날짜**: 2026-04-06
**의사결정자**: 3+1 에이전트 합의

---

## 맥락 (Context)

소프트웨어 개발에 코드 이외의 에셋(이미지, 사운드, 영상, 3D)이 필요하다. 생성 AI 모델은 빠르게 변하므로 고정 도구가 아닌 탐색 기반 선정이 필요하다.

## 결정 (Decision)

**Guide-First 에셋 파이프라인**: LLM이 최적 도구/모델을 탐색하여 안내하고, 사용자가 실행을 판단한다.

핵심 설계:
- 4카테고리 (IMAGE/AUDIO/VIDEO/3D), TEXT/DATA는 코드 파이프라인에서 처리
- 비AI 대안 우선 탐색 → 통합 허브(Replicate 등) → 개별 모델
- 안전장치: LLM 내장 거부 + 라이선스 표기 + 비용 경고
- 반복 상한 2회, MANIFEST는 JSON
- 폴백: AI → 대체 AI → 비AI(스톡/마켓/수동)

## 3+1 에이전트 합의 결과

| 에이전트 | 의견 | 핵심 근거 |
|---------|------|----------|
| Agent A (구현) | Guide-First 전환 필수 | Claude가 API 직접 실행 불가, WebFetch 불안정 |
| Agent B (품질) | 안전장치 + 라이선스 게이트 필수 | 콘텐츠 안전, 저작권 리스크 |
| Agent C (대안) | 허브 우선 + 비AI 대안 + 합의 폐지 | 과잉 설계 경고, 에셋은 재생성 비용 낮음 |
| **Reviewer** | **Guide-First + 4카테고리 + 허브 우선 + 최소 안전 세트** | **LLM-First/바이패스 원칙 일관 적용** |

## 결과 (Consequences)

- 긍정적: 실행 가능한 구체적 안내 제공, 모델 생태계 변화에 유연
- 긍정적: 비AI 대안 포함으로 최적 경로 보장
- 부정적: 사용자가 직접 실행해야 하는 부담
- 주의: Production 에셋은 라이선스 체크리스트 필수
