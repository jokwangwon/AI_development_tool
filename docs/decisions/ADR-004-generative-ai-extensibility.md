# ADR-004: 생성 AI 확장성 설계

**상태**: 승인 (3+1 합의 완료)
**날짜**: 2026-04-06
**의사결정자**: 3+1 에이전트 합의

---

## 맥락 (Context)

에셋 파이프라인(ADR-003)에 AI 학습/파인튜닝 지원, 모델 교체 용이성, 빠르게 변하는 생태계 대응이 필요하다.

## 결정 (Decision)

**외부 SDK 활용 + Config 기반 전환 + 학습 안내 가이드** 방식을 채택한다.

- 자체 어댑터 코드 계층: **제거** (Replicate/HF SDK가 이미 추상화 제공)
- 모델 교체: **generative-models.yaml 1개 파일** 수정으로 완료
- AI 학습: **Guide-First 안내** (자체 인프라 없음, 클라우드 서비스 안내)
- MVP: **Level 0(API 호출 안내)만**. Level 1(LoRA)은 필요 입증 후
- 데이터 거버넌스 체크리스트 필수

## 3+1 에이전트 합의 결과

| 에이전트 | 의견 | 핵심 근거 |
|---------|------|----------|
| Agent A | 어댑터 인터페이스 카테고리별 차이 너무 큼 | 단일 추상화 과도, 설정 2중화 문제 |
| Agent B | 데이터 거버넌스/안전성 전무 | 조건부 부적합, 체크리스트 필수 |
| Agent C | **자체 어댑터 전량 제거** | Guide-First에 코드 어댑터는 모순 |
| **Reviewer** | **C 채택 + B 안전장치 병합** | 외부 SDK 활용이 원칙 정합 |

## 근거 (Rationale)

초안의 자체 어댑터 계층(Python 인터페이스, 어댑터 파일, 레지스트리)은 Guide-First 원칙과 모순. 이미 Replicate/HuggingFace가 모델 추상화를 제공하므로, 이 프로젝트는 **어떤 SDK를 어떻게 쓰는지 안내하는 것**이 역할.

## 결과 (Consequences)

- 긍정적: 코드 유지보수 0, 모델 교체 = config 1줄 변경, 문서 70% 감축
- 긍정적: 외부 SDK 생태계 발전의 무료 수혜
- 부정적: 외부 SDK에 의존 (SDK 변경/중단 시 config 수정 필요)
- 주의: 학습 시 데이터 거버넌스 체크리스트 반드시 수행
