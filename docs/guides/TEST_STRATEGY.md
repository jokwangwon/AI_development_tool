# 테스트 전략 (Test Strategy)

> **SDD + TDD 기반 AI 자동화 개발 도구의 테스트 방법론**

**최종 수정**: 2026-04-06
**상위 문서**: `PROJECT_CONSTITUTION.md` 제2조

---

## 1. 테스트 원칙

### 1.1 TDD 사이클 (필수)

```
RED    → 실패하는 테스트 먼저 작성 (명세를 테스트로 표현)
GREEN  → 테스트를 통과하는 최소한의 코드 구현
REFACTOR → 코드 정리 (테스트는 계속 통과)
```

### 1.2 테스트는 명세의 실행 가능한 형태다

- SDD 문서의 명세 → 테스트 케이스로 변환
- 테스트가 통과하면 = 명세를 충족한다
- 테스트가 실패하면 = 명세와 코드가 불일치한다

## 2. 테스트 분류

### 2.1 테스트 피라미드

```
        /  E2E  \           적음, 느림, 비쌈
       /─────────\
      / Integration\        중간
     /───────────────\
    /    Unit Tests    \    많음, 빠름, 저렴
   /─────────────────────\
```

### 2.2 테스트 유형

| 유형 | 범위 | 속도 | Mock 사용 | 목적 |
|------|------|------|----------|------|
| **Unit** | 단일 함수/클래스 | <100ms | 외부 의존성만 | 로직 검증 |
| **Integration** | 모듈 간 연동 | <5s | 최소화 | 인터페이스 검증 |
| **E2E** | 전체 시스템 | <30s | 없음 | 시나리오 검증 |

### 2.3 에이전트 관련 테스트

| 대상 | 테스트 방법 | 검증 포인트 |
|------|-----------|------------|
| 에이전트 프롬프트 | 출력 형식 검증 | 구조화된 출력이 파싱 가능한가 |
| 하네스 Hook | Hook 실행 검증 | Hook이 올바른 시점에 실행되는가 |
| 3+1 합의 | 시나리오 테스트 | 합의 프로토콜이 정확히 작동하는가 |
| 도구 실행 | 입출력 검증 | 도구가 예상된 결과를 반환하는가 |

## 3. 커버리지 목표

| 영역 | 최소 목표 | 권장 |
|------|----------|------|
| 전체 | **70%** | 80% |
| 핵심 비즈니스 로직 | **90%** | 95% |
| 유틸리티 | 60% | 70% |
| 설정/구성 | 50% | 60% |

### 커버리지 원칙

- 숫자보다 **의미 있는 테스트**가 중요하다
- 100% 커버리지가 목표가 아니다
- 엣지 케이스와 실패 경로를 우선 테스트한다

---

## 4. 하네스 검증

### 4.1 피드백 루프 테스트

각 하네스 Layer가 올바르게 작동하는지 검증한다:

| Layer | 검증 방법 |
|-------|----------|
| Layer 1 (PostToolUse) | 의도적 lint 오류 삽입 → Hook 감지 확인 |
| Layer 2 (PreCommit) | 실패 테스트 포함 커밋 → 차단 확인 |
| Layer 3 (git hook) | 민감 정보 포함 → 차단 확인 |
| Layer 5 (3+1 합의) | 알려진 시나리오 → 기대 합의 결과 확인 |

### 4.2 에이전트 합의 품질 추적

| 메트릭 | 측정 | 목표 |
|--------|------|------|
| 합의율 | 3개 모두 동의한 비율 | > 60% |
| 유효 이견 비율 | 소수 의견 채택 비율 | > 10% |
| 누락 탐지율 | 1개만 발견한 CRITICAL | 추적 |

## 5. 테스트 실행

### 5.1 로컬 실행

```bash
# 전체 테스트
npm test          # 또는 pytest, gradle test 등 (기술 스택에 따라)

# 특정 테스트
npm test -- --grep "harness"

# 커버리지 리포트
npm test -- --coverage
```

### 5.2 CI 실행

- 모든 push에 전체 테스트 자동 실행
- PR 머지 조건: 모든 테스트 통과 + 커버리지 70% 이상
- 테스트 실패 시 머지 차단
