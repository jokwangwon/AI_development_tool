## 진행 사항

### 1. 게임 컨셉 도출 (3+1 멀티 에이전트 합의)

- 장르/테마 탐색: 시뮬레이션 → 관찰형 → 미니어처 갓 게임
- YouTube 미니어처 영상에서 영감 (거대한 손이 양동이로 물을 뿌리면 비가 내리는 장면)
- 3+1 합의 프로토콜 적용하여 컨셉 검증

**최종 컨셉**: "미니어처 갓 핸드"
- 테이블 위 미니어처 디오라마 세상
- 환경 토글 버튼(비/햇빛/바람/겨울) ON/OFF가 유일한 조작
- NPC가 자율적으로 생활, 신의 존재를 인식하지 않음
- 자연 다큐멘터리 같은 관찰 감성
- 시리즈/DLC 구조 (시대별 도구 변화)
- 여러 독립 디오라마 맵
- 자유 샌드박스 + 도감 수집

### 2. GDD (게임 설계 문서) 작성

- `docs/architecture/miniature-god-hand-gdd.md` 작성 완료
- 12개 섹션: 개요, 메카닉, NPC 시스템, 디오라마, 도감, 비주얼, UI, 저장, 기술스택, 로드맵, 리스크

### 3. 엔진 선택 (3+1 합의)

- Unity / UE5 / Godot / Custom / 기타 전체 비교
- Agent A(구현): Unity 1순위
- Agent B(안전성): Godot 1순위 (포기율 최저)
- Agent C(대안): Godot C# 1순위 (균형 최적)
- **최종 결정: Unity (C#)** — 사용자가 생태계/자료 풍부함 우선시, Python/JS 경험으로 C# 학습 부담 완화

### 4. 작업 환경 구성

- 노트북(macOS): Unity Editor (GUI 작업)
- 서버(Linux SSH via Tailscale): Claude Code (코드/설계)
- 연결: Git/GitHub
- 서버에 .NET 8.0 SDK 설치 완료

### 5. C# 기초 학습 (완료)

**Lesson 1** (`Program.cs`): Python/JS 비교 기반 기초 문법
- 변수/타입, 조건문, 반복문, 함수, 클래스, enum

**Lesson 2** (`Lesson2_UnityPatterns.cs`): Unity 핵심 패턴
- MonoBehaviour 라이프사이클 (Awake → Start → Update)
- 싱글톤 패턴 (TimeManager, WeatherManager)
- 이벤트/옵저버 패턴 (날씨 변경 → NPC 알림)
- FSM 실전 구현 (NPC 상태 전환)
- 게임 시뮬레이션 실행 (하루 사이클, 날씨 반응)
